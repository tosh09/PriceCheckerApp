# ANTIGRAVITY - プロジェクト憲法

## 1. プロジェクト概要
**名称**: Price Checker App (仮)
**目的**: せどりビジネスにおける店舗仕入効率化。既存アプリの古く使い勝手の悪いUIを刷新し、モダンで直感的なUXを提供する最新の仕入支援アプリを構築する。
**ターゲット**: せどりビジネスで店舗仕入れを行うユーザー（日本国内）
**利用シーン**: 店舗で気になる商品を見つけた際、スマホのカメラでバーコード等を読み取り、素早く利益が出るか判断する。

## 2. 開発戦略（MVP原則による段階的開発）
本プロジェクトは、確実に「動く体験」を最速で提供するため、以下の段階を踏んで開発を進める。

### Phase 1: コア機能の貫通テスト（超最小構成）
- UI（Vue.js/Bootstrap）のベース構築。
- カメラによるバーコード読み取り機能のプロトタイプ実装。
- バックエンド（Spring Boot）経由での簡単なモックデータ返却、またはKeepa APIを介した単一EC（Amazon）の簡易価格取得。
- ※まずは「アプリで読み込み → サーバと通信 → 画面に表示」の連携が成立することを目標とする。

### Phase 2: MVPの主要機能実装
- 楽天・Yahooへの情報取得拡張。
- 現在価格と最安値の取得。
- 仕入金額の手入力と、各手数料を考慮した利益計算ロジックの実装。

### Phase 3: アプリアドオン & DB永続化
- 履歴保存機能（位置情報、日時）の実装。
- 簡易DB（H2やSQLite等）の組込から、必要に応じたPostgreSQL等への移行検討。
- Google Play公開に向けたビルド・パッケージング最適化。

## 3. 技術スタック
- **フロントエンド / アプリ**: 
  - JavaScript, HTML, CSS
  - UIフレームワーク: Vue.js, Bootstrap (※要件に応じて最新のモダンUIへアプローチ)
  - ネイティブアプリ化: WebViewアプリ、または Capacitor 等を使ったハイブリッドアプリとしてGoogle Play向けに構築予定（要確認）
- **バックエンド**: 
  - 言語: Kotlin
  - フレームワーク: Spring Boot
  - インフラ: Cloudサーバ（展開先は未定）
- **データベース**: 初期段階は簡易DB、将来的にPostgreSQL
- **外部システム**: Keepa API (Amazon)、その他 各種ECサイトAPI

## 4. セキュリティ & ルール
- ユーザー認証: 不要（現時点）
- 「守りの三柱（技術的防御・法的・通信の安全）」をクリアするまで全世界公開（本番デプロイ）は行わない。
- データベースや重要ファイル（キーなど）を変更・構築する場合は必ず事前に確認を行う。
